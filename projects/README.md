# GitHub Portfolio Structure

This folder structure is ready to connect to GitHub and publish your work.

## What's Here

- `README.md` — Vault overview (this is the structure)
- `LINKING-GUIDE.md` — How to move work from internal → public
- `projects/` — Individual projects (code + docs)
- `portfolio/` — Your portfolio landing page
- `public-templates/` — Templates for projects and docs

## Next Steps

1. **Pick your first portfolio project** from your `storytelling-engine/projects/`
2. **Follow LINKING-GUIDE.md** to create a public version
3. **Start with one project** to establish the pattern
4. **Iterate** — Add more projects as you complete them

## Connecting to GitHub

This folder is ready to become your GitHub portfolio repository.

When you're ready:
1. Create a new GitHub repo (e.g., `portfolio`, `my-work`, or `engineering-portfolio`)
2. Clone to this location or add this folder as remote
3. Push your first projects
4. Link from your GitHub profile README

See LINKING-GUIDE.md for detailed instructions.
